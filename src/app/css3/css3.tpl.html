Curves & Rounded Corners

Rounded corners are a new, very useful feature. You can achieve them with the css class border-radius.

This is shown with different values in the images to the right. From left to right, the values are 10%, 30%, and 100%. The 100% is very useful for cropping an image into a circle without using any code.

Note that this property is not specific to images, and can be used for any container.


Individual Corners

border-radius is actually just a shorthand for four other CSS classes: border-top-left-radius, border-top-right-radius, border-bottom-right-radius, and border-bottom-left-radius.

Use these as needed for some nifty effects.


Backgrounds & Scaling

Full page backgrounds are a standard feature of the modern web. There are a few things to note here.

First, to set your background image, you'll need to set background to url(...)

Second, background-size is very important for maintaining consistent backgrounds. It can take several values, but the ones to look out for in responsive design are:

  • contain: the background image will scale up, as long as the entire iamge fits within the container and retains it's proportions. This means it may not fill the entire container. background: url(...) no-repeat can be useful in this case.
  • cover: The background image will cover the entire container and retain its proportions. Some parts of the image will not be visible if it has a different aspect ratio than the container.

Colors

CSS3 has also introduced some new color possibilities. If you've grown accustomed to using rgb colors, there's no need to change what you're doing. HSL is just a different way to produce the same colors.

Opacity ranges from 0.0 (completely invsible) to 1.0 (completely opaque).

  • RGBA: typical RGB format, but with alpha (also known as opacity or transparency). rgba(255,0,0,0.1)
  • HSL: Hue, saturation, and lightness. hsl(120, 100%, 50%)
  • HSLA: The same as HSL, but with an extra parameter for opacity.

opacity is also a standalone CSS property that will affect the entire container.

When text is overflowing its parent container

text-overflow is very useful for this. When text is too big for a container, it can either clip it (clip) or clip it and leave an ellipses (ellipsis). For both of these, you're going to want to also set overflow to hidden.

Another important thing to consider is word-wrap: break-word, which will break words apart if they are extending out of their parent container.

Regarding breaking, word-break is handy for this. break-all will let words be broken apart at any point, while keep-all will prevent words from breaking.

Fonts

Fonts are important! If you want to utilize a font that a user doesn't have installed, font-faces are for you.

Supported formats are .TFF, .OTF, .WOFF, and .EOT.

To utilize this, first define your font-face:

@font-face {
   font-family: customFont;
   src: url(...);
}

And then:

div { font-family: customFont; }


Spinning out of control

CSS3 brings us a bunch of 2D and 3D transformations. rotate is the most conventionally useful.

Set transform to rotate(90deg) to utilize this.

transform: skew(20deg, 20deg) is an option as well, if you ever need that.

The two arguments are for the x- and y-axis.
